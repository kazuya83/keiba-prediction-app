# 実行計画 19: ドキュメント生成と開発者ガイド整備

## ゴール
- API/データ/運用に関するドキュメントを自動生成・整備し、開発者とユーザーが参照できる状態を作る。

## 手順
1. OpenAPI仕様を `backend/app/main.py` からエクスポートし、`docs/api/openapi.json` を生成するスクリプトを用意。
2. `docs/` ディレクトリを作成し、`mkdocs` or `docusaurus` でドキュメントサイトを構築。CIでビルド＆デプロイ（`gh-pages` 等）。
3. データベースER図を `alembic` もしくは `sqlacodegen` から生成し、`docs/db/` に保存。
4. 機械学習パイプラインの使用方法・再学習手順を `docs/ml/README.md` にまとめる。
5. 運用Runbook（ジョブ再実行手順、通知設定、ログ確認方法）を `docs/runbook.md` に作成。
6. フロントエンドのデザインガイドライン（カラーパレット、コンポーネント命名）を `docs/ui/style-guide.md` に記述。
7. CIにドキュメントビルドジョブを追加し、破損リンクチェック（`lychee` など）を実行。失敗時に通知。

## 成果物
- `docs/` 配下に整理されたMarkdown/サイト、および自動生成スクリプト。
- 最新OpenAPI/ER図/Runbook/MLガイド。
- CIでのドキュメントビルド・リンクチェック結果。

## 依存・前提
- APIエンドポイント、DBスキーマ、MLパイプラインが実装済み。
- CI環境が静的サイトをビルド・公開できる。

## リスクと対策
- **リスク**: ドキュメント更新漏れ。  
  **対策**: PRテンプレートにドキュメント更新チェック項目を追加。
- **リスク**: 自動生成スクリプトの失敗に気づかない。  
  **対策**: CIで失敗時に通知し、破損リンクを検知。*** End Patch
